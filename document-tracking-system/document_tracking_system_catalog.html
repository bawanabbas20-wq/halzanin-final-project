QR code flow
Citizen submits → DB creates application with tracking_code = 'TRK-'.strtoupper(Str::random(8))
QR code generated via QrCode::format('svg')->size(200)->generate(route('track', $code))
Citizen downloads PDF with QR card (barryvdh/laravel-dompdf)
Scanning the QR opens /track/{code} — shows current status timeline

Sticky forms (session persistence)
This is your key differentiator. Implement it with Laravel sessions + Livewire or plain JS autosave.
On every field change, POST to /appointment/save-draft which stores data in session(['draft' => $request->all()])
On page load, blade populates value="{{ session('draft.field_name', '') }}"
On final submit, clear draft from session

I'm building a Laravel 10 web app: a Digital Public Document Tracking System for the Passport Directorate in Kurdistan. Common mistakes to avoid
Always define $fillable in every Model — never use $guarded = [] (mass assignment risk)
Use Storage::disk('local')->put() for documents, not public disk — keep files private
Use Form Request classes for validation, not inline $request->validate() in every controller
Don't use Laravel Passport — use Breeze sessions (no API needed for this app)
Add UNIQUE index on tracking_code column in migration, not just in code
